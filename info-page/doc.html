Building from Source

NOTE: MathLex is distributed in a precompiled, optimized, and minified JS file. Building MathLex from source should not be necessary unless you know what you are doing and need/want to make changes.

Source Code

Requirements and Dependencies

Make sure you have the following applications and libraries installed:

  • Node.js with the following modules: CoffeeScript, Handlebars, Jison, and UglifyJS
  • Ruby with the Compass gem.

Build Instructions

Download and extract the MathLex source code. MathLex is written in CoffeeScript files located in the src directory. These files are easier to read and understand than their compiled JavaScript counterparts.

Type the following into your console to run the automated build script:

% cd /path/to/mathlex-source
% make

The compiled JS files will be written to build/browser. Individual components will be saved to the build directory and mimic the structure of the CoffeeScript files

How MathLex Works

When provided with a valid MathLex string, MathParser.parse() produces an abstract syntax tree (AST) representing the explicit, inferred value of the MathLex code. This representative tree is built from different "node" types represented as a recursive array. Every node has at least one element (i.e. the 0th element or node[0] in JS) that is a string indicator of the type of node. Appendix B lists all possible nodes and their structure.

Under the hood, MathLex has two components: a Tokenizer and a Parser. The Tokenizer is responsible for translating the characters in the MathLex input string into a list of Tokens, a way to group related characters into a single symbol. For example, "<=" is shorthand for "less than or equal to" (in display math, '≤') and is comprised of two separate characters. The Tokenizer groups these characters into a TLessEqual for the parser.

The Parser then reads the list of tokens and assembles the corresponding abstract syntax tree (AST) based on the rules in src/grammar.coffee. The parser code itself is generated by Jison and for most purposes is unreadable.
